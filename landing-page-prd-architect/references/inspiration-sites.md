# Inspiration Sites

## Design Galleries

### Dribbble
- **URL:** https://dribbble.com/
- **Busca:** `https://dribbble.com/search/[termo]`
- **Termos úteis:**
  - `landing page`
  - `website design`
  - `hero section`
  - `[segmento] landing page` (ex: saas landing page)
  - `[estilo] website` (ex: minimal website)

### Awwwards
- **URL:** https://www.awwwards.com/
- **Filtros:** websites, nominees, elements
- **Melhor para:** Sites premiados, alta qualidade

### Behance
- **URL:** https://www.behance.net/
- **Busca:** `web design`, `landing page`, `ui design`
- **Melhor para:** Cases completos, processo de design

### Land-book
- **URL:** https://land-book.com/
- **Melhor para:** Só landing pages, organizado por categoria

### Landingfolio
- **URL:** https://www.landingfolio.com/
- **Melhor para:** Templates e inspirações de LP

### One Page Love
- **URL:** https://onepagelove.com/
- **Melhor para:** Single page websites

### SaaS Landing Page
- **URL:** https://saaslandingpage.com/
- **Melhor para:** Especificamente SaaS

### Lapa Ninja
- **URL:** https://www.lapa.ninja/
- **Melhor para:** Landing pages curadas

### Mobbin
- **URL:** https://mobbin.com/
- **Melhor para:** Mobile UI patterns

### Screenlane
- **URL:** https://screenlane.com/
- **Melhor para:** UI flows e screens

## Workflow de Pesquisa

1. Definir segmento/estilo do projeto
2. Buscar no Dribbble: `[segmento] landing page`
3. Buscar no Awwwards por categoria
4. Verificar Land-book para mais exemplos
5. Documentar 3-5 referências principais
6. Identificar padrões comuns
7. Definir o que adaptar de cada um
